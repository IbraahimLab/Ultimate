How It Works

From prompt to verified patch

  1. 01

    Scan + Context Build

    Agent builds project index and loads memory + policy settings.

  2. 02

    Structured Plan

    Model returns JSON with plan, tool actions, verify commands, and optional memory updates.

  3. 03

    Action + Diff Approval

    Tools execute, diffs are shown, writes require manual approval.

  4. 04

    Verify + Repair

    Tests/lint/format run, stack traces parsed, retries continue until done or stopped.

Implementation Examples

Use it as CLI, JS SDK, or Python SDK

Node / TypeScript SDK

import { CodingAgent, loadRuntimeConfig } from "ultimate-vibe-agent";

const config = loadRuntimeConfig(process.cwd(), {
  maxIterations: 8,
  model: "moonshotai/kimi-k2-instruct-0905",
  baseUrl: "https://api.groq.com/openai/v1",
});

const agent = new CodingAgent(config);

const ui = {
  info: console.log,
  warn: console.warn,
  error: console.error,
  ask: async (q: string) => "continue",
  confirm: async (q: string) => true,
};

await agent.runTask(
  "refactor auth middleware and add tests for edge cases",
  ui,
);

Python SDK Wrapper

from vibe_agent_sdk import VibeAgentClient

client = VibeAgentClient(
    executable="vibe-agent",
    env={
        "GROQ_API_KEY": "your-key",
        "VIBE_MODEL": "moonshotai/kimi-k2-instruct-0905",
        "VIBE_BASE_URL": "https://api.groq.com/openai/v1",
    },
)

result = client.run_task(
    "create an API retry utility and tests",
    max_iterations=6,
    check=False,
)

print(result.returncode)
print(result.stdout)

Quick Start

Get running in under 2 minutes

  1. Set provider env in `.env` (`GROQ_API_KEY`, model, base URL).
  2. Run `npm install` and `npm run dev`.
  3. Give a task in English. Approve diffs before writes.
  4. Use `/config` to inspect active model, limits, and state dir.

.env example

GROQ_API_KEY=your_groq_api_key_here
VIBE_MODEL=moonshotai/kimi-k2-instruct-0905
VIBE_BASE_URL=https://api.groq.com/openai/v1
VIBE_MAX_ITERATIONS=6
VIBE_AUTO_VERIFY=true
VIBE_STATE_DIR=.vibe-agent

Policy and Safety

Guardrails built for team usage

Permission Policy

Block dangerous command patterns and protected file paths from writes.

Secret Detection

Detects common credential patterns before write operations are applied.

Audit Trail

Every task loop can be traced via JSONL events in `.vibe-agent/audit`.

Policy file sample (`.vibe-agent/policy.json`)

{
  "allowRunCommand": true,
  "allowWrite": true,
  "allowedCommandPrefixes": [],
  "blockedCommandPatterns": [
    "rm\\\\s+-rf\\\\s+/",
    "curl\\\\s+.+\\\\|\\\\s*sh\\\\b"
  ],
  "blockedWriteGlobs": [
    ".env",
    ".env.*",
    "**/.env",
    "**/.env.*",
    ".git/**"
  ],
  "allowPotentialSecrets": false
}

FAQ

Common setup questions

Do I need a PyPI token?

No, if you use PyPI Trusted Publishing (OIDC). If not configured, use API token publishing.

Can I use a model provider other than OpenAI?

Yes. The SDK uses OpenAI-compatible endpoints. Groq + Kimi is already configured.

Where does memory live?

In `.vibe-agent/memory.json` by default. Change path with `VIBE_STATE_DIR`.

How do I prevent dangerous commands?

Set command rules in `.vibe-agent/policy.json` (`allowedCommandPrefixes`, `blockedCommandPatterns`).
